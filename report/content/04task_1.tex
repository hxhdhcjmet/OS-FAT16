\section{任务一}

\subsection{任务要求}

本任务要求实现一个简单的类 FAT-16 文件系统，并支持目录创建、目录查看、文件删除、文件打开、文件读取、文件写入和文件关闭等基本功能。实验中不使用真实磁盘，而是通过定义内存数组：

\begin{lstlisting}[language=C]
#define DISK_MAXLEN 2560
char Disk[DISK_MAXLEN];
\end{lstlisting}

将该数组模拟为一块硬盘空间，在此基础上完成文件系统的数据组织和文件操作。

根据实验要求，本文件系统只需要考虑单进程访问场景，因此暂不需要处理多个进程同时访问文件或目录时的同步问题。实验重点在于设计并实现文件系统的核心数据结构，包括超级块、FAT 表、目录项和数据块等内容。其中，FAT 表用于维护文件所占数据块之间的链接关系，目录项用于记录文件名、文件类型、起始块号和文件大小等元信息，数据块则用于保存具体的文件内容或目录内容。

通过本任务，需要完成以下基本操作：

\begin{itemize}
    \item \texttt{mkdir}：在当前目录下创建新的子目录；
    \item \texttt{ls}：显示当前目录下的文件和子目录信息；
    \item \texttt{delete}：删除指定文件或空目录，并释放其占用的数据块；
    \item \texttt{open}：打开指定文件，返回文件描述符；
    \item \texttt{read}：根据文件描述符读取文件内容；
    \item \texttt{write}：向指定文件写入数据，并更新 FAT 表和目录项；
    \item \texttt{close}：关闭已经打开的文件。
\end{itemize}

因此，本任务的核心目标是通过内存模拟的方式，完成一个具备基本目录管理、文件读写和空间回收能力的简化文件系统，并通过运行结果验证各项文件操作的正确性。

\subsection{实验过程}

\subsubsection{磁盘空间划分与整体设计}

本任务使用内存数组 \texttt{Disk[DISK\_MAXLEN]} 模拟硬盘空间。由于实验中定义的磁盘总大小为 2560 字节，因此本实验将磁盘划分为若干个固定大小的数据块，每个数据块大小设置为 128 字节，总共可以划分为 20 个块。

\begin{lstlisting}[language=C]
#define DISK_MAXLEN 2560
#define BLOCK_SIZE 128
#define BLOCK_NUM  (DISK_MAXLEN / BLOCK_SIZE)
\end{lstlisting}

在此基础上，文件系统的磁盘布局设计如下：

\begin{itemize}
    \item 第 0 块：超级块，用于保存文件系统的基本信息；
    \item 第 1 块：FAT 表，用于记录各数据块的使用状态和链接关系；
    \item 第 2 块：根目录区，用于保存根目录下的目录项；
    \item 第 3 块到第 19 块：数据区，用于保存文件内容或子目录内容。
\end{itemize}

其中，FAT 表是本实验实现类 FAT-16 文件系统的核心结构。每个 FAT 表项对应一个数据块，其取值含义如下：

\begin{itemize}
    \item \texttt{FAT\_FREE}：表示该块空闲；
    \item \texttt{FAT\_END}：表示该块已经被占用，并且是文件或目录的最后一个块；
    \item 其他块号：表示当前块的下一个数据块编号。
\end{itemize}

因此，当一个文件只占用一个数据块时，对应 FAT 表项为 \texttt{FAT\_END}；当一个文件跨越多个数据块时，FAT 表会形成一条链式结构，例如 \texttt{4 -> 5 -> 6 -> END}。

\subsubsection{文件系统基本数据结构设计}

为了保存文件系统的基本信息，本实验定义了超级块结构体，用于记录块大小、块数量、FAT 表位置、根目录位置和数据区起始位置等信息。

\begin{lstlisting}[language=C]
typedef struct {
    int block_size;
    int block_num;
    int fat_start;
    int root_dir_block;
    int data_start;
} SuperBlock;
\end{lstlisting}

同时，为了描述文件和目录，本实验定义了目录项结构体。每个目录项记录一个文件或子目录的基本信息，包括名称、是否使用、类型、起始块号和文件大小。

\begin{lstlisting}[language=C]
typedef struct {
    char name[NAME_LEN];
    unsigned char used;
    unsigned char type;
    unsigned short first_block;
    unsigned short size;
    char reserved[10];
} DirEntry;
\end{lstlisting}

其中，\texttt{type} 字段用于区分普通文件和目录，\texttt{first\_block} 字段表示文件或目录内容在数据区中的起始块号，\texttt{size} 字段用于记录文件大小。通过目录项和 FAT 表的配合，系统可以根据文件名找到其起始块，并沿 FAT 链读取或释放文件内容。

\subsubsection{文件系统初始化}

文件系统初始化由 \texttt{fs\_format()} 函数完成。该函数首先清空整个 \texttt{Disk} 数组，然后初始化超级块信息，并将 FAT 表全部设置为空闲状态。随后，将第 0 块、第 1 块和第 2 块分别标记为已占用，因为它们分别用于超级块、FAT 表和根目录。

初始化后的 FAT 表状态如下：

\begin{itemize}
    \item \texttt{Block 0: END}，表示超级块已占用；
    \item \texttt{Block 1: END}，表示 FAT 表已占用；
    \item \texttt{Block 2: END}，表示根目录已占用；
    \item \texttt{Block 3} 到 \texttt{Block 19} 均为 \texttt{FREE}，表示数据区尚未分配。
\end{itemize}

运行结果中，初始目录为空，FAT 表中只有前三个块被占用，其余块均为空闲状态，说明文件系统格式化过程正确。

\subsubsection{数据块分配与释放}

为了支持文件和目录的创建，本实验实现了数据块分配函数 \texttt{alloc\_block()}。该函数从数据区起始块开始扫描 FAT 表，寻找空闲块。如果找到空闲块，则将其 FAT 表项设置为 \texttt{FAT\_END}，并清空该块内容，最后返回该块编号。

文件删除或覆盖写入时，需要释放原文件占用的数据块。因此，本实验实现了 \texttt{free\_chain()} 函数。该函数从文件起始块开始，沿 FAT 表记录的链式结构逐块向后遍历，并将经过的块重新设置为 \texttt{FAT\_FREE}，从而完成空间回收。

这两个函数是后续 \texttt{mkdir}、\texttt{write} 和 \texttt{delete} 操作的基础。其中，\texttt{alloc\_block()} 负责从空闲空间中分配数据块，\texttt{free\_chain()} 负责在删除文件时回收文件占用的全部数据块。

\subsubsection{目录项查找与目录创建}

为了在目录中查找文件或子目录，本实验实现了目录项查找函数 \texttt{find\_entry()} 和空目录项查找函数 \texttt{find\_empty\_entry()}。其中，\texttt{find\_entry()} 根据文件名在当前目录中查找已有目录项，\texttt{find\_empty\_entry()} 用于寻找尚未使用的目录项位置。

在实现 \texttt{mkdir} 操作时，系统首先检查当前目录下是否已经存在同名目录或文件。如果不存在，则调用 \texttt{alloc\_block()} 分配一个新的数据块作为子目录的数据块，然后在当前目录中写入新的目录项。目录项中记录目录名、目录类型、起始块号和大小等信息。

实验中执行：

\begin{lstlisting}[language=C]
fs_mkdir(ROOT_BLOCK, "dir1");
\end{lstlisting}

运行结果显示：

\begin{lstlisting}
mkdir success: dir1, block=3
[DIR ] dir1       first=3 size=0
\end{lstlisting}

同时，FAT 表中 \texttt{Block 3} 由 \texttt{FREE} 变为 \texttt{END}。这说明系统成功为目录 \texttt{dir1} 分配了第 3 块，并在根目录中建立了对应目录项。

\subsubsection{文件打开与关闭}

为了管理已经打开的文件，本实验设计了一个简单的打开文件表 \texttt{open\_table}。每个打开文件表项记录文件是否被占用、所在目录块、文件名和打开模式等信息。

\begin{lstlisting}[language=C]
typedef struct {
    int used;
    int dir_block;
    char filename[NAME_LEN];
    char mode;
} OpenFile;
\end{lstlisting}

当执行 \texttt{open} 操作时，系统根据文件名在当前目录中查找文件。如果以读模式打开，则文件必须已经存在；如果以写模式打开，则当文件不存在时，系统会自动创建新的文件目录项。随后，系统在打开文件表中寻找空闲表项，并返回对应的文件描述符。

当执行 \texttt{close} 操作时，系统只需要将打开文件表中对应表项的 \texttt{used} 字段清零，表示该文件描述符已经释放。

实验中，文件 \texttt{a.txt} 被成功打开，并返回文件描述符 \texttt{fd=0}，说明打开文件表能够正常管理文件打开状态。

\subsubsection{文件写入与读取}

文件写入由 \texttt{fs\_write()} 实现。本实验采用覆盖写的方式，即每次写入文件时，如果文件原来已经占用数据块，则先调用 \texttt{free\_chain()} 释放原有 FAT 链，然后根据新写入内容的长度重新分配数据块，并将这些数据块通过 FAT 表连接起来。

当写入内容长度小于一个块大小时，文件只需要占用一个数据块。例如实验中向 \texttt{a.txt} 写入字符串：

\begin{lstlisting}[language=C]
fs_write(fd, "Hello FAT File System!");
\end{lstlisting}

运行结果显示：

\begin{lstlisting}
[FILE] a.txt      first=4 size=22
Block 4: END
\end{lstlisting}

这说明 \texttt{a.txt} 的起始块为第 4 块，文件大小为 22 字节。由于该文件内容小于一个块大小，所以只占用一个数据块，FAT 表中 \texttt{Block 4} 被标记为 \texttt{END}。

文件读取由 \texttt{fs\_read()} 实现。读取时，系统首先根据文件描述符找到对应目录项，再根据目录项中的 \texttt{first\_block} 找到文件起始块，并沿 FAT 链依次读取文件内容。实验运行结果显示：

\begin{lstlisting}
read result: Hello FAT File System!
\end{lstlisting}

这说明系统能够根据目录项和 FAT 表正确定位文件数据，并成功读取文件内容。

文件删除由 \texttt{fs\_delete()} 实现。删除普通文件时，系统首先在当前目录中查找目标文件目录项。如果文件存在，则根据目录项中的起始块号调用 \texttt{free\_chain()} 释放其占用的所有数据块，最后清空对应目录项。

实验中删除 \texttt{a.txt} 后，运行结果显示：

\begin{lstlisting}

Block 4: FREE
\end{lstlisting}

这说明文件删除后，\texttt{a.txt} 原本占用的第 4 块已经被重新标记为空闲状态，文件系统完成了空间回收。该结果表明，本实验实现的 \texttt{delete} 操作不仅删除了目录项，而且正确释放了文件数据块。

本实验使用内存数组 \texttt{Disk} 模拟一块容量为 2560 字节的硬盘，并以固定大小的数据块作为磁盘空间分配的基本单位。相关宏定义如下：

typedef struct {

其中，\texttt{block\_size} 和 \texttt{block\_num} 分别记录数据块大小和数据块总数，\texttt{fat\_start} 表示 FAT 表所在位置，\texttt{root\_dir\_block} 表示根目录所在块，\texttt{data\_start} 表示数据区的起始块号。

在文件系统初始化过程中，第 0、1、2 块会被标记为已经占用，其余数据块均处于空闲状态。这样的划分方式虽然比真实 FAT-16 文件系统更加简化，但已经能够支持本实验要求的目录管理、文件读写和空间回收功能。


\subsubsection{FAT 表设计}

FAT 表是本实验实现类 FAT-16 文件系统的核心数据结构。每个 FAT 表项与一个磁盘块一一对应，用于记录该数据块当前是否空闲，以及该数据块之后是否还连接其他数据块。

本实验使用 \texttt{unsigned short} 类型保存 FAT 表项，并定义了两种特殊状态：

\begin{lstlisting}[language=C]
#define FAT_FREE 0x0000
#define FAT_END  0xFFFF
\end{lstlisting}

不同 FAT 表项取值的含义如下：

\begin{itemize}
    \item \texttt{FAT\_FREE}：表示当前数据块尚未分配，可以被新文件或目录使用；
    \item \texttt{FAT\_END}：表示当前数据块已经被占用，并且是文件或目录占用的最后一个块；
    \item 其他块号：表示当前数据块之后连接的下一个数据块编号。
\end{itemize}

例如，当一个文件只占用第 4 块时，其 FAT 表项为：

\begin{center}
    \texttt{Block 4: END}
\end{center}

当一个文件占用多个数据块时，FAT 表会形成链式结构。例如，一个文件依次占用第 4、5、6 块，则 FAT 表中的记录为：

\begin{center}
    \texttt{Block 4: 5}
    
    \texttt{Block 5: 6}
    
    \texttt{Block 6: END}
\end{center}

对应的数据块链接关系可以表示为：

\begin{center}
    \texttt{Block 4} $\rightarrow$ \texttt{Block 5}
    $\rightarrow$ \texttt{Block 6}
    $\rightarrow$ \texttt{END}
\end{center}

为了支持数据块的动态管理，本实验实现了 \texttt{alloc\_block()} 和 \texttt{free\_chain()} 两个辅助函数。

\texttt{alloc\_block()} 从第 3 块开始扫描 FAT 表，寻找状态为 \texttt{FAT\_FREE} 的数据块。找到空闲块后，将其标记为 \texttt{FAT\_END}，清空对应块中的数据，并返回该块编号。

\texttt{free\_chain()} 用于释放文件占用的数据块。该函数从文件起始块开始，根据 FAT 表项逐块向后遍历，将经过的所有数据块重新设置为 \texttt{FAT\_FREE}，直到遇到 \texttt{FAT\_END} 为止。该函数主要用于文件删除和覆盖写入操作。

通过上述设计，文件的数据块不需要在磁盘中连续分布，只需要由 FAT 表维护块之间的链接关系即可。


\subsubsection{目录项设计}

目录用于保存文件和子目录的基本信息。本实验将目录同样视为一种特殊的数据内容，每个目录占用一个数据块，目录块中保存若干个目录项。

目录项结构体定义如下：

\begin{lstlisting}[language=C]
typedef struct {
    char name[NAME_LEN];
    unsigned char used;
    unsigned char type;
    unsigned short first_block;
    unsigned short size;
    char reserved[10];
} DirEntry;
\end{lstlisting}

各字段含义如下：

\begin{itemize}
    \item \texttt{name}：记录文件名或目录名；
    \item \texttt{used}：记录当前目录项是否已经被使用；
    \item \texttt{type}：区分普通文件和目录；
    \item \texttt{first\_block}：记录文件内容或子目录内容所在的起始块号；
    \item \texttt{size}：记录普通文件的大小；
    \item \texttt{reserved}：预留字段，用于保证目录项结构具有较为规整的大小。
\end{itemize}

为了区分普通文件和目录，本实验定义了两种目录项类型：

\begin{lstlisting}[language=C]
#define TYPE_FILE 1
#define TYPE_DIR  2
\end{lstlisting}

例如，创建目录 \texttt{dir1} 后，根目录中会新增如下目录项：

\begin{center}
    \texttt{[DIR ] dir1 \quad first=3 \quad size=0}
\end{center}

其中，\texttt{first=3} 表示目录 \texttt{dir1} 占用第 3 块，\texttt{size=0} 表示该目录初始为空。

创建文件 \texttt{a.txt} 并写入数据后，根目录中会新增如下目录项：

\begin{center}
    \texttt{[FILE] a.txt \quad first=4 \quad size=22}
\end{center}

其中，\texttt{first=4} 表示文件内容从第 4 块开始存储，\texttt{size=22} 表示文件大小为 22 字节。

为了实现目录项管理，本实验还实现了两个辅助函数：

\begin{itemize}
    \item \texttt{find\_entry()}：根据文件名或目录名，在当前目录中查找对应目录项；
    \item \texttt{find\_empty\_entry()}：在当前目录中寻找尚未使用的目录项，用于创建新文件或新目录。
\end{itemize}

本实验采用简化设计，每个目录占用一个固定大小的数据块。该设计能够满足本次实验对目录创建、目录查看和文件管理的基本要求。


\subsubsection{基本文件操作实现}

在完成磁盘划分、FAT 表和目录项设计后，本实验实现了 \texttt{mkdir}、\texttt{ls}、\texttt{delete}、\texttt{open}、\texttt{read}、\texttt{write} 和 \texttt{close} 等基本文件操作。

\paragraph{1. 创建目录}

目录创建功能由 \texttt{fs\_mkdir()} 实现。该函数首先检查当前目录中是否已经存在同名文件或目录。如果不存在，则调用 \texttt{alloc\_block()} 分配新的数据块，并在当前目录中寻找空闲目录项。随后，将目录名、目录类型和起始块号写入目录项。

其基本流程如下：

\begin{enumerate}
    \item 检查当前目录中是否已经存在同名目录项；
    \item 在当前目录中寻找空闲目录项；
    \item 从 FAT 表中分配一个空闲数据块；
    \item 初始化新目录对应的数据块；
    \item 在当前目录中写入新目录项。
\end{enumerate}

\paragraph{2. 显示目录内容}

目录查看功能由 \texttt{fs\_ls()} 实现。该函数遍历当前目录块中的所有目录项，并输出已经使用的目录项信息，包括文件类型、文件名、起始块号和文件大小。

输出格式如下：

\begin{lstlisting}
[DIR ] dir1       first=3 size=0
[FILE] a.txt      first=4 size=22
\end{lstlisting}

\paragraph{3. 打开与关闭文件}

为了记录已经打开的文件，本实验定义了打开文件表：

\begin{lstlisting}[language=C]
typedef struct {
    int used;
    int dir_block;
    char filename[NAME_LEN];
    char mode;
} OpenFile;
\end{lstlisting}

执行 \texttt{fs\_open()} 时，系统根据文件名查找目录项，并在打开文件表中寻找空闲表项。如果使用读模式打开文件，则目标文件必须已经存在；如果使用写模式打开文件，而目标文件尚不存在，则系统会自动创建新的文件目录项。

文件打开成功后，函数返回文件描述符 \texttt{fd}。执行 \texttt{fs\_close()} 时，只需要清空对应打开文件表项，即可释放该文件描述符。

\paragraph{4. 写入文件}

文件写入功能由 \texttt{fs\_write()} 实现。本实验采用覆盖写策略：如果目标文件原本已经占用数据块，则首先调用 \texttt{free\_chain()} 释放原有 FAT 链，然后根据新写入数据的长度重新分配数据块。

写入过程中，系统会将数据按照每块 128 字节进行拆分，并通过 FAT 表将多个数据块连接起来。如果写入内容小于或等于 128 字节，则只需要分配一个数据块；如果写入内容大于 128 字节，则需要分配多个数据块，并形成 FAT 链。

\paragraph{5. 读取文件}

文件读取功能由 \texttt{fs\_read()} 实现。该函数首先根据文件描述符找到对应文件目录项，然后根据目录项中的 \texttt{first\_block} 找到文件起始块。读取过程中，系统根据 FAT 表逐块向后遍历，直到读取完指定长度的数据，或者到达 FAT 链末尾。

\paragraph{6. 删除文件或目录}

删除功能由 \texttt{fs\_delete()} 实现。删除普通文件时，系统首先根据目录项中的 \texttt{first\_block} 找到文件起始块，然后调用 \texttt{free\_chain()} 释放文件占用的所有数据块，最后清空对应目录项。

对于目录，本实验只允许删除空目录。如果目标目录中仍然存在文件或子目录，则拒绝执行删除操作，从而避免目录结构出现不一致。


\subsubsection{运行结果与分析}

完成上述功能后，本实验依次对文件系统初始化、目录创建、文件写入、文件读取、文件删除以及跨块文件写入进行了测试。

\paragraph{1. 文件系统初始化}

首先调用 \texttt{fs\_format()} 对模拟磁盘进行格式化。运行结果显示，第 0、1、2 块均被标记为 \texttt{END}，第 3 块至第 19 块均被标记为 \texttt{FREE}。

\begin{lstlisting}
format success
Block  0: END
Block  1: END
Block  2: END
Block  3: FREE
...
Block 19: FREE
\end{lstlisting}

这说明超级块、FAT 表和根目录区已经被正确初始化，数据区处于空闲状态。

\paragraph{2. 创建目录}

随后，在根目录中创建目录 \texttt{dir1}。运行结果如下：

\begin{lstlisting}
mkdir success: dir1, block=3
[DIR ] dir1       first=3 size=0
Block  3: END
\end{lstlisting}

结果表明，系统成功为目录 \texttt{dir1} 分配了第 3 块，并在根目录中建立了对应目录项。FAT 表中第 3 块由 \texttt{FREE} 变为 \texttt{END}，说明目录创建过程正确。

\paragraph{3. 文件写入与读取}

接下来，以写模式打开文件 \texttt{a.txt}，并向其中写入字符串 \texttt{Hello FAT File System!}。运行结果如下：

\begin{lstlisting}
open success: a.txt, fd=0, mode=w
write success: fd=0, size=22 bytes
close success: fd=0
[FILE] a.txt      first=4 size=22
Block  4: END
\end{lstlisting}

由于写入内容长度为 22 字节，小于一个数据块大小，因此文件只占用第 4 块。随后，以读模式重新打开该文件，能够正确读取文件内容：

\begin{lstlisting}
open success: a.txt, fd=0, mode=r
read success: fd=0, size=22 bytes
read result: Hello FAT File System!
close success: fd=0
\end{lstlisting}

这说明 \texttt{open}、\texttt{write}、\texttt{read} 和 \texttt{close} 功能均能够正常运行。

\paragraph{4. 文件删除与空间回收}

删除文件 \texttt{a.txt} 后，运行结果如下：

\paragraph{5. 跨块文件写入}

为了进一步验证 FAT 表的链式管理机制，本实验创建了文件 \texttt{big.txt}，并向其中写入 299 字节的数据。由于一个数据块只能保存 128 字节，因此该文件需要占用 3 个数据块。

运行结果如下：

\begin{lstlisting}
open success: big.txt, fd=0, mode=w
write success: fd=0, size=299 bytes
close success: fd=0
[FILE] big.txt    first=4 size=299
Block  4: 5
Block  5: 6
Block  6: END
\end{lstlisting}

从 FAT 表可以看出，文件 \texttt{big.txt} 的数据块链接关系为：

\begin{center}
    \texttt{Block 4} $\rightarrow$ \texttt{Block 5}
    $\rightarrow$ \texttt{Block 6}
    $\rightarrow$ \texttt{END}
\end{center}

随后，以读模式打开 \texttt{big.txt}，能够正确读取文件内容：

\begin{lstlisting}
open success: big.txt, fd=0, mode=r
read success: fd=0, size=299 bytes
read big.txt first part:
ABCDEFGHIJKLMNOPQRSTUVWXYZABCDEFGHIJKLMNOPQRSTUVWXYZ...
close success: fd=0
\end{lstlisting}

该结果表明，本实验不仅能够正确读写单块文件，还能够为跨块文件建立 FAT 链，并沿 FAT 链完成文件内容读取。至此，任务一要求的目录创建、目录查看、文件删除、文件打开、文件读取、文件写入和文件关闭等功能均已实现。
